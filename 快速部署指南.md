## 🔧 部署步驟

### 步驟 1: 部署 Google Apps Script（10分鐘）

1. 在 Google Sheets「輔具盤點系統」中
2. 點選「擴充功能」→「Apps Script」
3. 刪除預設的 `Code.gs` 內容
4. **複製本資料夾中的 `Code_v2.gs` 完整內容**並貼上
5. 點選「檔案」→「儲存」
6. 點選「部署」→「新增部署作業」
7. 選擇「網頁應用程式」
8. 設定：
   - **執行身分**：我
   - **具有存取權的使用者**：任何人
9. 點選「部署」
10. **複製部署網址**（非常重要！）

**網址格式：**
```
https://script.google.com/macros/s/AKfycbx.../exec
```

---

### 步驟 2: 設定前端（2分鐘）

1. 開啟本資料夾中的 `config.js`
2. 將部署網址填入：

```javascript
API_URL: 'https://script.google.com/macros/s/你的網址/exec',
```

3. 儲存檔案

---

### 步驟 3: 部署前端（選擇一種方式）

#### 方式 A: 使用 Netlify（推薦，最簡單）

1. 註冊 [Netlify](https://www.netlify.com)（免費）
2. 拖曳整個 `ptatinventory-git` 資料夾內容到 Netlify
3. 等待部署完成（約 1 分鐘）
4. 獲得永久網址（如：`https://your-app.netlify.app`）

#### 方式 B: 使用 Vercel

1. 註冊 [Vercel](https://vercel.com)（免費）
2. 上傳 `frontend` 資料夾
3. 部署完成

#### 方式 C: 使用 GitHub Pages

1. 建立 GitHub Repository
2. 上傳 `frontend` 資料夾內的所有檔案
3. 啟用 GitHub Pages
4. 使用提供的網址

## 🔍 疑難排解

### 問題 1: 無法載入人員清單
**原因**：人員清單工作表沒有資料
**解決**：
1. 檢查 Google Sheets「人員清單」工作表
2. 確認至少有一筆人員資料且狀態為「在職」

### 問題 2: 掃描條碼後找不到輔具
**原因**：財產編號不存在或格式錯誤
**解決**：
1. 確認輸入的是「財產編號」（如：1A0448080400001）
2. 系統也支援「庫存編號」（如：T10800000459）
3. 檢查 Google Sheets 是否有此編號

### 問題 3: 照片上傳失敗
**原因**：Google Drive 權限或空間不足
**解決**：
1. 檢查 Google Drive 是否有足夠空間
2. 確認 Apps Script 已授權存取 Drive

### 問題 4: 區域篩選不正確
**原因**：放置地點欄位缺少區域資訊
**解決**：
1. 檢查「放置地點」欄位格式
2. 正確格式：`輔具中心-屏北區:地下室展示輔具區`
3. 必須包含「屏中區」或「屏北區」

### 問題 5: 自動判斷需清潔/需充電不正確
**原因**：輔具品名格式問題
**解決**：
1. 系統判斷邏輯：品名包含「電動」→需充電
2. 其他→需清潔
3. 可在 `config.js` 調整關鍵字
