# 輔具盤點系統 - 部署指南

## 📋 部署前準備

### 1. 建立 Google Sheets 資料庫

1. 開啟 [Google Sheets](https://sheets.google.com)
2. 建立新試算表，命名為「輔具盤點系統」
3. 建立以下四個工作表（Sheet）：

#### 工作表 1: 輔具清單
在第一列輸入以下標題：
```
輔具編號 | 輔具名稱 | 類別 | 區域 | 管理人員 | 擺放位置 | 狀態 | 最後盤點日期 | 最後盤點人員 | 備註 | 照片網址 | 建立日期 | 更新日期
```

#### 工作表 2: 外借記錄
在第一列輸入以下標題：
```
外借編號 | 輔具編號 | 輔具名稱 | 外借單位 | 聯絡人 | 聯絡電話 | 外借日期 | 預計歸還日期 | 實際歸還日期 | 外借狀態 | 經辦人員 | 簽名圖片 | 備註 | 建立日期
```

#### 工作表 3: 人員清單
在第一列輸入以下標題：
```
人員編號 | 姓名 | 區域 | 電話 | Email | 權限 | 狀態
```

**重要：請先新增人員資料，例如：**
```
STAFF-001 | 王小明 | 屏中區 | 0912-345-678 | wang@example.com | 管理員 | 在職
STAFF-002 | 李小華 | 屏北區 | 0923-456-789 | lee@example.com | 一般使用者 | 在職
```

#### 工作表 4: 盤點記錄
在第一列輸入以下標題：
```
記錄編號 | 輔具編號 | 盤點日期 | 盤點人員 | 盤點方式 | 原擺放位置 | 新擺放位置 | 原狀態 | 新狀態 | 備註
```

## 🚀 部署步驟

### 步驟 1: 部署 Google Apps Script

1. 在 Google Sheets 中，點選上方選單「**擴充功能**」→「**Apps Script**」
2. 刪除預設的 `Code.gs` 內容
3. 複製本資料夾中的 `Code_v2.gs` 完整內容並貼上
4. 點選「**檔案**」→「**儲存**」，命名為「輔具盤點系統」
5. 點選上方「**部署**」→「**新增部署作業**」
6. 點選「**選取類型**」→「**網頁應用程式**」
7. 設定如下：
   - **說明**：輔具盤點系統 v1.0
   - **執行身分**：我
   - **具有存取權的使用者**：任何人
8. 點選「**部署**」
9. 授權應用程式（首次需要授權）
10. **複製部署的網址**（非常重要！）

**網址格式範例：**
```
https://script.google.com/macros/s/AKfycbx.../exec
```

---

### 步驟 2: 設定前端

1. 開啟本資料夾中的 `config.js`
2. 將剛才複製的部署網址貼到 `API_URL`：

```javascript
const CONFIG = {
  API_URL: 'https://script.google.com/macros/s/AKfycbx.../exec',
  // ... 其他設定
};
```

3. 儲存檔案

---

### 步驟 3: 部署 Web App

#### 方法 A: 使用 Google Apps Script 內建 Web App（推薦）

1. 在 Apps Script 編輯器中，點選「**檔案**」→「**新增**」→「**HTML**」
2. 命名為 `index`
3. 複製 `index.html` 的內容並貼上
4. 重複步驟 1-3，依序新增以下 HTML 檔案：
   - `styles` (複製 `styles.css` 內容，但要用 `<style>` 標籤包起來)
   - `app` (複製 `app.js` 內容，但要用 `<script>` 標籤包起來)
   - `config` (複製 `config.js` 內容，但要用 `<script>` 標籤包起來)
   - `camera` (複製 `camera.js` 內容，但要用 `<script>` 標籤包起來)
   - `signature` (複製 `signature.js` 內容，但要用 `<script>` 標籤包起來)

5. 修改 `index.html`，將外部 CSS/JS 引用改為內嵌或使用 `<?!= HtmlService.createHtmlOutputFromFile('檔名').getContent() ?>`

**簡化方式：將所有 CSS 和 JS 直接內嵌到 index.html**

#### 方法 B: 使用免費託管服務（Netlify/Vercel）

1. 註冊 [Netlify](https://www.netlify.com) 或 [Vercel](https://vercel.com)
2. 將本資料夾前端檔案拖曳到部署區域
3. 部署完成後，會獲得一個網址（例如：`https://your-app.netlify.app`）

#### 方法 C: 使用 GitHub Pages

1. 建立 GitHub 帳號並新增 Repository
2. 上傳 `frontend` 資料夾內的所有檔案
3. 在 Repository 設定中啟用 GitHub Pages
4. 使用提供的網址存取

## 📱 安裝為手機 App（PWA）

### iOS (iPhone/iPad)

1. 使用 Safari 開啟系統網址
2. 點選下方「**分享**」按鈕
3. 選擇「**加入主畫面**」
4. 輸入名稱「輔具盤點」
5. 點選「**加入**」
6. 在主畫面會出現 App 圖示

### Android

1. 使用 Chrome 開啟系統網址
2. 點選右上角「**⋮**」選單
3. 選擇「**加入主畫面**」或「**安裝應用程式**」
4. 輸入名稱「輔具盤點」
5. 點選「**加入**」
6. 在主畫面會出現 App 圖示

## 🔧 進階設定

### 啟用條碼掃描功能

系統預設使用手動輸入條碼。若要啟用相機掃描條碼：

1. 在 `index.html` 的 `<head>` 區塊加入：
```html
<script src="https://unpkg.com/@zxing/library@latest"></script>
```

2. 若需替換掃描套件，請直接調整 `app.js` 中的 `Html5Qrcode` 掃描流程

### 自訂 App 圖示

1. 準備兩張圖片：
   - `icon-192.png` (192x192 像素)
   - `icon-512.png` (512x512 像素)
2. 放在 `frontend` 資料夾
3. 圖示建議使用輔具相關圖案

### 設定 Google Drive 資料夾權限

1. 開啟 Google Drive
2. 找到「輔具照片」和「外借簽名」資料夾
3. 右鍵 → 「共用」
4. 設定為「知道連結的任何人都可以查看」

## 📊 資料匯入

### 從現有 AppSheet 匯出資料

1. 開啟 AppSheet App
2. 匯出資料為 CSV 或 Excel
3. 開啟 Google Sheets「輔具清單」工作表
4. 點選「檔案」→「匯入」
5. 上傳 CSV/Excel 檔案
6. 選擇「附加到目前工作表」
7. 確認欄位對應正確

### 批次新增輔具資料

可以直接在 Google Sheets 中複製貼上資料，格式如下：

```
EQ-2024-001 | 輪椅 | 行動輔具 | 屏中區 | 王小明 | 1樓倉庫A區 | 可用 | | | | | 2024-01-01 |
EQ-2024-002 | 助行器 | 行動輔具 | 屏北區 | 李小華 | 2樓倉庫B區 | 可用 | | | | | 2024-01-01 |
```

## 🔒 安全性設定

### 限制存取權限

1. 在 Apps Script 部署設定中，將「具有存取權的使用者」改為「僅限我的組織」
2. 這樣只有您組織內的 Google 帳號可以存取

### 備份資料

建議定期備份 Google Sheets：
1. 點選「檔案」→「下載」→「Microsoft Excel (.xlsx)」
2. 儲存到安全的位置

## ❓ 疑難排解

### 問題 1: 無法載入人員清單
**解決方法：**
- 確認 Google Sheets 中「人員清單」工作表已建立
- 確認至少有一筆人員資料
- 檢查工作表名稱是否正確（必須是「人員清單」）

### 問題 2: 掃描條碼後沒有反應
**解決方法：**
- 確認輔具編號已存在於「輔具清單」中
- 檢查條碼編號格式是否正確
- 查看瀏覽器 Console 是否有錯誤訊息

### 問題 3: 照片上傳失敗
**解決方法：**
- 確認已授權相機權限
- 檢查網路連線
- 確認 Google Drive 有足夠空間

### 問題 4: 手機無法安裝為 App
**解決方法：**
- 確認使用 HTTPS 連線
- iOS 必須使用 Safari 瀏覽器
- Android 必須使用 Chrome 瀏覽器

### 問題 5: API 呼叫失敗
**解決方法：**
- 確認 `config.js` 中的 `API_URL` 正確
- 重新部署 Apps Script
- 檢查 Apps Script 授權是否完成
