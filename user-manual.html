輔具盤點管理系統使用說明書

本說明書提供同仁進行掃描盤點、我的輔具管理、外借與歸還、月報表下載、照片維護與批次盤點時的完整操作流程。若系統有更新，可直接替換本 HTML 檔並重新部署。

適用對象：中心同仁
文件形式：單一 HTML
可獨立部署到其他 repo

章節目錄

1. 系統概觀 2. 登入與首頁 3. 掃描盤點 4. 輔具清單與詳細資料 5. 我的輔具與批次盤點 6. 外借與歸還 7. 月報表 8. 盤點判定規則 9. 常見問題 10. 文件部署建議

1. 系統概觀

系統分成五個主要頁面：掃描盤點、輔具清單、外借管理、月報表、我的輔具。其中盤點、外借、照片更新、詳細資料修改等動作，都會影響該輔具當月是否被視為已完成盤點工作。

掃描盤點用手機相機掃描財產編號，快速完成單筆盤點。
輔具清單查看全體輔具，點卡片可開詳細資料。
我的輔具查看自己管理的輔具，支援單筆與批次盤點。
外借管理進行外借、歸還，以及處理歷史外借補登歸還。

2. 登入與首頁

  1. 開啟系統後，從下拉選單選擇自己的姓名。
  2. 按下 登入系統 後，系統會依照目前人員資料帶入可用功能。
  3. 登入成功後，右上方會顯示當前登入者名稱，並可點選登出。
  4. 頁首的 使用說明 會開啟本說明書。
若登入名單沒有自己的姓名，請先確認 Google Sheets 的人員清單工作表是否已建檔，且狀態為在職。

3. 掃描盤點

標準流程

  1. 進入 掃描盤點。
  2. 按下 開始掃描，允許相機權限。
  3. 對準財產編號條碼，系統會自動辨識並完成盤點。
  4. 成功後會顯示輔具品名與財產編號。
  5. 可選擇 繼續掃描 或 檢視輔具。

檢視輔具

若掃描完成後想補拍照片、補確認資料、或直接進入詳細資料頁，可按下 檢視輔具。系統會打開該輔具的詳細資料頁。

若掃描後顯示查無資料，代表該財編不在目前展示輔具清單中，可能尚未建檔或不屬於本系統管理項目。

4. 輔具清單與詳細資料

在 輔具清單 頁面可依區域與目前動態篩選輔具，也可以用關鍵字搜尋財產編號、庫存編號、品名、保管人、放置地點與備註。

點卡片後可查看

  • 輔具照片
  • 放置地點、保管人、目前動態
  • 來源、原始編號、入庫日期、最後盤點時間
  • 當次作為與備註

可編輯欄位

欄位說明
保管人可直接改成目前實際管理人員。
放置地點反映目前實際擺放位置。
當次作為可勾選需清潔、需充電或同時勾選兩者。
備註可記錄補充說明。
詳細資料頁上方的照片區若顯示「尚無照片（點我拍照）」，可直接拍攝並上傳新照片。

5. 我的輔具與批次盤點

我的輔具 只顯示目前登入者負責管理的輔具，適合進行日常巡檢與大量盤點。

單筆盤點

  • 每張卡片都保留 完成盤點 按鈕。
  • 點卡片本體可開詳細資料，不影響單筆盤點流程。

批次盤點

  1. 先用搜尋或 已盤點 / 未盤點 tab 篩選目標清單。
  2. 點選 批次盤點模式。
  3. 勾選要盤點的多筆輔具。
  4. 可使用 全選目前篩選結果 一次勾選目前畫面中的所有資料。
  5. 按下 批次完成盤點 後，系統會再次確認。
  6. 完成後會顯示成功與失敗筆數，並更新卡片狀態。
批次盤點只會處理目前登入人員負責管理的輔具，不會跨到全體輔具清單。

6. 外借與歸還

外借

  1. 進入 外借管理。
  2. 可用掃描或手動輸入財產編號選取輔具。
  3. 填寫借用人、借用起始日期時間、預計歸還日期時間、用途等資料。
  4. 借用人簽名後送出。

歸還

  1. 可掃描或手動輸入財產編號查詢外借中的輔具。
  2. 填寫實際歸還日期時間並由中心人員簽名。
  3. 若是舊資料顯示為外借中但沒有完整外借明細，系統可補登直接歸還。

7. 月報表

月報表頁面可選擇區域與月份，並指定報表類型。

  • 盤點月報：產出當月盤點／維護清冊 PDF。
  • 外借月報：產出當月外借紀錄 PDF。

報表目前會儲存至 Google Drive 的 輔具盤點系統 / 月報表PDF 資料夾。

8. 盤點判定規則

系統目前採用「當月只要對輔具做過任一有效作業，就視為本月已完成盤點工作」的邏輯。有效作業包含：

  • 掃描或單筆快速盤點
  • 批次盤點
  • 外借
  • 歸還
  • 拍攝或更新輔具照片
  • 從詳細資料頁儲存維護相關欄位
系統已改採專用的「最近作業時間」欄位，不再直接用一般更新日期判斷，以避免匯入資料或舊修改造成誤判。

9. 常見問題

掃不到條碼

請確認相機權限已允許，並避免逆光。若現場不易掃描，可改用我的輔具卡片直接盤點。

照片看不到

請確認照片已上傳成功，系統會自動把 Google Drive 圖片網址轉成可顯示格式。若仍看不到，可重新拍攝一次。

批次盤點有部分失敗

通常代表該筆資料已不屬於目前登入者管理，或財編資料異常。請回到卡片逐筆確認。

月報表產不出來

請先確認該月份是否真的有資料，並確認 GAS 已部署最新版且前端 API_URL 指向正確網址。

10. 文件部署建議

這份說明書是單一 HTML，可直接獨立部署到另一個 repo，例如 GitHub Pages、Netlify 或 Vercel。

建議做法

  1. 建立一個專門放說明文件的新 repo。
  2. 只放這個 user-manual.html 與需要的靜態資產。
  3. 部署完成後取得公開網址。
  4. 回到主系統的 config.js，將 MANUAL_URL 改成該公開網址。
這樣做很好，因為說明書可以獨立更新、單獨部署、也不會和主系統 API 發版綁在一起。
